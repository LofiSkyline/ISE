# Lab 4: AI Model Fairness Testing - Workflow Notes

本文档记录了 `lab4/demo.py` 的工作流程及其涉及的关键公平性测试概念。

## 1. 核心流程图

1. **加载数据** (CSV) -> 2. **准备特征与标签** -> 3. **加载预训练模型** (.h5) -> 4. **执行推理** -> 5. **计算群体公平性指标** -> 6. **执行个体公平性测试**

## 2. 详细步骤说明

### 2.1 加载与准备 (Steps 1-3)
- **数据源**: 使用 `pandas` 读取 `dataset/processed_adult.csv`。
- **特征提取**: 将最后一列 `Class-label` 作为目标标签，其余列作为输入特征 `X`。
- **模型加载**: 使用 `keras.models.load_model` 加载位于 `DNN/` 目录下的 H5 格式预训练模型。

### 2.2 公平性指标计算 (Step 5 - Group Fairness)
程序计算了两个经典的群体公平性（Group Fairness）指标。在理解公式前，先看符号定义：

- **$Y$**: **真实标签 (Ground Truth)**。即数据集中记录的真实情况（例如：这个人的真实年收入是否真的超过 50k）。
- **$\hat{Y}$ (读作 Y-hat)**: **模型预测值**。即 AI 模型根据特征计算后，给出的判断结果。
- **$Y=1$**: 代表正向结果（如：高收入、准予贷款、通过考试）。
- **$P(...|...)$**: **条件概率**。即“在某种条件下，发生某事的概率”。

#### 指标 1：统计偏差 (Statistical Parity Difference, SPD)
- **公式**: $P(\hat{Y}=1 | 	ext{非特权组}) - P(\hat{Y}=1 | 	ext{特权组})$
- **通俗解释**: “非特权组被预测为高收入的概率” 减去 “特权组被预测为高收入的概率”。
- **核心逻辑**: 它**不看**这个人的真实能力（不看 $Y$），只看**结果的分布**。
- **理想状态**: 如果两个概率相等（差值为 0），说明模型在给结果时，完全没有看性别这个标签。

#### 指标 2：机会平等偏差 (Equal Opportunity Difference, EOD)
- **公式**: $P(\hat{Y}=1 | Y=1, 	ext{非特权组}) - P(\hat{Y}=1 | Y=1, 	ext{特权组})$
- **通俗解释**: “在那些**真实收入确实高**的人群中，非特权组被模型正确找出来的概率” 减去 “特权组被模型正确找出来的概率”。
- **核心逻辑**: 它引入了真实标签 $Y=1$ 作为前提。这实际上是在比较不同群体之间的**真阳性率 (True Positive Rate/Recall)**。
- **直观理解**: 如果两个组里都有很有才华的人，模型是否对某一类人“视而不见”？如果差值为负，说明模型更容易忽略非特权组中的优秀人才。

### 2.3 个体公平性测试 (Step 6 - Individual Fairness)
- **原理**: 如果两个个体仅在敏感属性（如性别、种族）上有所不同，而其他属性相同，那么公平的模型应该给出相同的预测。
- **操作**: 
    1. 复制原始数据。
    2. 将敏感属性（本例中为 `gender`）的值翻转（0 变 1，1 变 0）。
    3. 重新进行推理。
    4. 统计预测结果发生改变的样本数量。
- **结论**: 如果预测结果改变，说明模型在该样本上存在个体不公平性。

## 3. 特权组 (Privileged) vs. 非特权组 (Unprivileged)

在公平性机器学习中，这两个概念用于区分在社会或历史背景下受到不同待遇的群体：

- **特权组 (Privileged Group)**: 指在数据集中通常处于有利地位、获得正向结果概率较高的群体。在本实验的 Adult 数据集中，通常指**男性 (gender=1)**。
- **非特权组 (Unprivileged Group)**: 指在历史上或系统中可能受到歧视、获得正向结果概率较低的群体。在本实验中指**女性 (gender=0)**。

**为什么要区分？**
公平性测试的目标是检查模型是否对“非特权组”存在偏见。例如，如果模型更倾向于给男性（特权组）预测高收入，而忽视了具有相同能力的女性（非特权组），则认为模型是不公平的。

## 4. Adult 数据集字段说明

Adult 数据集（又称 Census Income 数据集）的目标是预测一个人的年收入是否超过 $50,000。

| 字段名 | 说明 | 示例值 (已处理) |
| :--- | :--- | :--- |
| **race** | 种族 | 4 (White), 2 (Black) 等 |
| **gender** | 性别 | 1 (Male), 0 (Female) |
| **age** | 年龄 | 39, 50, 38 等 |
| **hours-per-week** | 每周工作时长 | 40, 13, 20 等 |
| **education** | 受教育程度 | 9 (Bachelors), 11 (HS-grad) 等 |
| **occupation** | 职业 | 0 (Admin), 3 (Exec-managerial) 等 |
| **Class-label** | **目标标签** | 0 (<=50K), 1 (>50K) |

> **注意**: 该数据集中的分类变量（如职业、种族）已经过数值化处理（Label Encoding），以便模型读取。

## 5. 实验关键变量 (修正)
- **敏感属性 (Sensitive Attribute)**: `gender` (性别)。
- **特权组 (Privileged Value)**: `1` (通常指男性)。
- **非特权组 (Unprivileged Value)**: `0` (通常指女性)。
- **目标标签 (Target Label)**: `Class-label` (预测年收入是否超过 50k)。
